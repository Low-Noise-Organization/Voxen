Preguntas Frecuentes

Respuestas a las preguntas más comunes sobre Voxen, su arquitectura y uso.

¿Qué es Voxen exactamente?

Voxen es un orquestador de builds y capa de supply chain para Docker. Proporciona una CLI única y consistente para construir, empaquetar, firmar, generar SBOM, dockerizar y publicar proyectos de cualquier lenguaje. No compite con Docker — lo potencia.

¿Voxen reemplaza a Maven, Gradle, npm o cargo?

No. Voxen delega en tus build tools existentes. Cuando ejecutas voxen build, detecta automáticamente si usas Maven, Gradle, npm, cargo, etc. y lo invoca. Voxen es una capa de orquestación que unifica la interfaz, no un reemplazo de los build tools.

¿Voxen compite con Docker?

No. Voxen se sienta delante de Docker. Docker se encarga del runtime aislado y portable. Voxen se encarga de construir el artefacto, generar la imagen Docker óptima (con distroless, SBOM, checksums, firma) y publicarla. Voxen necesita Docker para funcionar. Son herramientas complementarias.

¿Qué lenguajes soporta Voxen?

Voxen soporta 6 lenguajes de forma nativa mediante RuntimePlugin: Java, Kotlin, Node.js, Python, Go, Rust y .NET. Cada runtime tiene su propia imagen base Docker óptima. Se pueden añadir más lenguajes implementando la interfaz RuntimePlugin.

¿Qué hace el comando voxen dockerize?

Genera una imagen Docker a partir del artefacto ya construido. Pero no es un docker build cualquiera: genera automáticamente un Dockerfile óptimo (multi-stage si es necesario, con distroless como base), copia el SBOM, las firmas y los checksums dentro de la imagen, añade 15+ labels OCI y Voxen con toda la trazabilidad del build, y opcionalmente publica la imagen en un registry, la firma con Cosign, y ejecuta un linter de calidad.

¿Puedo publicar en repositorios privados?

Sí. Voxen soporta 6 publicadores: local, Maven Central, Artifactory, Nexus, S3 y Docker (Docker Hub, GHCR, ECR, GCR). Las credenciales se guardan en ~/.voxen/credentials.json o mediante variables de entorno (*_TOKEN, *_HOST).

¿Cómo maneja Voxen múltiples entornos?

Voxen tiene 3 perfiles integrados (dev, staging, prod) con variables y settings preconfigurados. También puedes crear perfiles personalizados con voxen-{nombre}.json. Usa el flag -p / --profile para seleccionar el entorno.

¿Es seguro para producción?

Sí. Voxen incluye: firma GPG de artefactos, checksums SHA-256/512/MD5, SBOM CycloneDX, firma Cosign de imágenes Docker, escaneo de vulnerabilidades Trivy, SLSA provenance, sandboxing de plugins, auditoría estructurada de todas las operaciones, y generación de imágenes distroless (mínima superficie de ataque).

¿Cómo habilito la salida JSON para CI/CD?

Añade --json a cualquier comando. Voxen producirá JSON estructurado por stdout y suprimirá los mensajes legibles por humanos. Cada comando tiene su propio esquema JSON documentado en la referencia CLI.

¿Qué versión de Java necesito?

Voxen requiere Java 21 o superior. Usa features modernas como records, pattern matching, y el HttpClient nativo. Para mejor rendimiento, compila con GraalVM native-image y consigue un binario con inicio en milisegundos.

¿Cómo habilito el autocompletado en terminal?

Ejecuta ./gradlew generateCompletions para generar scripts de completado para bash y zsh, luego actívalos: source build/completions/voxen_completion.bash (bash) o source build/completions/voxen_completion.zsh (zsh).

¿Dónde se guardan los logs y la configuración?

Audit logs del proyecto: .voxen/audit.jsonl. Credenciales de usuario: ~/.voxen/credentials.json. Telemetría local: ~/.voxen/telemetry.jsonl. Configuración del proyecto: voxen.json en la raíz.

¿Cómo contribuyo a Voxen?

Voxen es open source bajo licencia Apache 2.0. Puedes contribuir reportando issues, enviando pull requests, escribiendo plugins (runtime, build, o publishers), o mejorando la documentación. Visita el repositorio de GitHub para las guías de contribución.

¿Voxen soporta multi-arquitectura?

Sí. El comando voxen dockerize --platform linux/amd64,linux/arm64 usa Docker Buildx para generar imágenes multi-arquitectura. Voxen configura automáticamente QEMU y buildx si es necesario.

¿Puedo usar Voxen sin Docker?

Sí. Los comandos voxen init, build, package, publish (a repositorios tradicionales) y deploy funcionan sin Docker. Solo voxen dockerize y voxen publish:docker requieren Docker instalado.
