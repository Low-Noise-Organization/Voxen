Inicio Rápido

Crea un proyecto, constrúyelo, dockerízalo y publícalo en menos de 5 minutos.

1 Crear un Proyecto

mkdir mi-app && cd mi-app

# Inicializar con Voxen
voxen init -n mi-app -r java -t jar -o build

Esto crea un archivo voxen.json en el directorio actual:

{
  "name": "mi-app",
  "runtime": "java",
  "target": "jar",
  "output": "build"
}
✅ Archivo validado automáticamente contra JSON Schema (draft 2020-12). Si falta name o runtime, Voxen te muestra un error claro con sugerencias.

2 Añadir Archivos de Build

Voxen delega la compilación a tu build tool existente. Crea un pom.xml (Maven) o build.gradle.kts (Gradle):

Maven (pom.xml)

<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.ejemplo</groupId>
  <artifactId>mi-app</artifactId>
  <version>1.0.0</version>
  <packaging>jar</packaging>
</project>

Gradle (build.gradle.kts)

plugins { id("java-library") }
group = "com.ejemplo"
version = "1.0.0"

3 Construir

voxen build

Voxen detecta automáticamente Maven o Gradle y ejecuta el build. Cada operación queda registrada en el audit log (.voxen/audit.jsonl).

4 Empaquetar

voxen package

Produce el artefacto distribuible. Automáticamente se generan:

  • Checksums SHA-256, SHA-512 y MD5
  • Firma GPG (detach-sign .asc)
  • SBOM CycloneDX 1.6 JSON
Package created: /home/usuario/mi-app/build/mi-app-1.0.0.jar
Format: jar
Size: 4.2 KB

Checksums:
  SHA-256: a1b2c3... ✓
  SHA-512: d4e5f6... ✓
  MD5:     g7h8i9... ✓
Signed: GPG key ABCD1234 ✓
SBOM: /app/bom.json ✓

5 Dockerizar

Genera una imagen Docker óptima y mínima con toda la metadata de supply chain:

voxen dockerize --tag ghcr.io/usuario/mi-app:1.0.0 --lint
✅ Imagen generada con:
• Base distroless (sin shell, sin package manager)
• SBOM CycloneDX en /app/bom.json
• Labels OCI estándar (created, version, revision, source, title)
• Labels Voxen (build-id, profile, runtime, checksums)
• Firma GPG como sidecar

Inspecciona la imagen:

docker inspect ghcr.io/usuario/mi-app:1.0.0 --format '{{json .Config.Labels}}' | jq

6 Publicar Multi-Arquitectura

# Publicar para amd64 + arm64, firmado y firmado con Cosign
voxen dockerize --tag ghcr.io/usuario/mi-app:1.0.0 \
  --push \
  --platform linux/amd64,linux/arm64 \
  --sign --lint --json

Salida JSON para integrar con CI/CD:

{"status":"success","project":"mi-app","tag":"ghcr.io/usuario/mi-app:1.0.0","dockerfile":"/workspace/Dockerfile","pushed":true}

7 Desplegar con Perfiles

# Desplegar a desarrollo
voxen deploy --profile dev

# Desplegar a staging con salida verbose
voxen deploy --profile staging --verbose

# Desplegar a producción (usa perfil integrado que salta tests)
voxen deploy --profile prod --json

Los perfiles integrados ajustan variables y settings automáticamente:

PerfilVariablesSettings
devenv=development, debug=trueverbose=true, skipTests=false
stagingenv=staging, optimization=standardverbose=false, skipTests=false
prodenv=production, optimization=aggressiveverbose=false, skipTests=true

Pipeline CI/CD Completo

Workflow listo para usar en GitHub Actions:

# .github/workflows/dockerize.yml (ya incluido en el proyecto)
name: Build & Dockerize
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with: { java-version: '21', distribution: 'temurin' }
      - run: ./gradlew build
      - run: |
          voxen build --profile prod --json
          voxen dockerize \
            --tag ghcr.io/${{ github.repository }}:latest \
            --push --platform linux/amd64,linux/arm64 \
            --lint --sign --json

O usa la acción reutilizable incluida:

- uses: ./.github/actions/voxen-build-push
  with:
    profile: prod
    tag: ghcr.io/${{ github.repository }}:${{ github.ref_name }}
    push: true
    sign: true
