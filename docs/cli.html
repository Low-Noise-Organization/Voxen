Referencia de Comandos CLI

Todos los comandos, opciones y formatos de salida de Voxen.

Opciones Globales

Estas opciones están disponibles en todos los comandos:

-v, --verbose     Salida detallada
    --debug       Salida de depuración (más detalle que verbose)
    --json        Salida JSON para automatización (CI/CD)
-p, --profile     Perfil de entorno: dev, staging, prod, o personalizado (default: dev)
    --help        Muestra ayuda del comando
    --version     Muestra la versión

Comandos

voxen init

Inicializa un nuevo proyecto Voxen creando voxen.json.

Opciones:
  -n, --name <name>        Nombre del proyecto (default: nombre del directorio)
  -r, --runtime <runtime>  Runtime: java, kotlin, rust, node, python, go, dotnet (default: java)
  -t, --target <format>    Formato destino: jar, war, zip, tar.gz, wheel
  -o, --output <dir>       Directorio de salida (default: dist)

Argumentos:
  [directorio]              Directorio del proyecto (default: directorio actual)

Ejemplos:
  voxen init -n mi-app -r java -t war -o build
  voxen init --json

voxen build

Construye el proyecto usando el build tool detectado (Maven, Gradle, npm, cargo, etc.).

Argumentos:
  [directorio]              Directorio del proyecto (default: directorio actual)

Ejemplos:
  voxen build
  voxen build --verbose
  voxen build --json
  voxen build -p staging

voxen package

Empaqueta el proyecto en un formato distribuible (JAR, ZIP, wheel, etc.). Genera checksums, firma GPG y SBOM automáticamente.

Argumentos:
  [directorio]              Directorio del proyecto (default: directorio actual)

Ejemplos:
  voxen package
  voxen package --json

voxen dockerize NUEVO

Genera una imagen Docker óptima desde el artefacto construido. Incluye automáticamente SBOM, checksums, firma y labels OCI.

Opciones:
  -t, --tag <tag>           Tag de la imagen (ej. ghcr.io/user/app:1.0.0) (obligatorio)
      --push                 Publicar la imagen en el registry tras construir
      --platform <lista>    Plataformas destino (ej. linux/amd64,linux/arm64)
      --native               Usar entrypoint nativo (GraalVM native-image)
      --lint                 Ejecutar linter de imagen tras construir
      --sign                 Firmar la imagen con Cosign (keyless)

Argumentos:
  [directorio]              Directorio del proyecto (default: directorio actual)

Ejemplos:
  voxen dockerize --tag ghcr.io/user/app:1.0.0
  voxen dockerize --tag ghcr.io/user/app:1.0.0 --push --platform linux/amd64,linux/arm64
  voxen dockerize --tag ghcr.io/user/app:1.0.0 --sign --lint --json
💡 La imagen generada es mínima. Usa distroless como base (sin shell, sin package manager). Incluye SBOM, checksums SHA-256/512/MD5, firma GPG y 15+ labels de trazabilidad.

voxen publish

Publica el artefacto en un repositorio configurado (local, Maven Central, Artifactory, Nexus, S3).

Argumentos:
  [directorio]              Directorio del proyecto (default: directorio actual)

Ejemplos:
  voxen publish
  voxen publish --json

voxen publish:docker NUEVO

Publica una imagen Docker en un registry (Docker Hub, GHCR, ECR, GCR).

Opciones:
  -t, --tag <tag>           Tag local a publicar (obligatorio)
  -r, --registry <url>      Registry destino (ej. ghcr.io/user) (obligatorio)
  -n, --name <name>        Nombre de la imagen (obligatorio)
      --version <version>  Tag de versión (default: latest)
      --platform <lista>   Plataformas para multi-arquitectura

Ejemplos:
  voxen publish:docker --tag app:1.0 --registry ghcr.io/user --name mi-app
  voxen publish:docker --tag app:1.0 --registry ghcr.io/user --name mi-app \
    --platform linux/amd64,linux/arm64

voxen deploy

Despliega el proyecto en un entorno (dev, staging, prod).

Argumentos:
  [directorio]              Directorio del proyecto (default: directorio actual)

Ejemplos:
  voxen deploy
  voxen deploy --profile staging
  voxen deploy --profile prod --json

voxen info

Muestra información del sistema, plugins instalados y estado del proyecto.

Argumentos:
  [directorio]              Directorio del proyecto (default: directorio actual)

Ejemplos:
  voxen info
  voxen info --json

Formato de Salida JSON

Todos los comandos soportan --json para integración con CI/CD. Cada comando tiene su propio esquema:

init

{"status":"success","project":"mi-app","runtime":"java","path":"/home/user/mi-app/voxen.json"}

build

{"status":"success","project":"mi-app","durationMs":3250,"artifacts":["build/mi-app-1.0.jar"],"output":"build"}

package

{"status":"success","project":"mi-app","path":"build/mi-app-1.0.jar","format":"jar","sizeBytes":4096}

dockerize

{"status":"success","project":"mi-app","tag":"ghcr.io/user/app:1.0.0","dockerfile":"/workspace/Dockerfile","pushed":true}

publish

{"status":"success","project":"mi-app","repository":"maven-central","url":"https://repo1.maven.org/..."}

deploy

{"status":"success","project":"mi-app","environment":"staging","url":"https://staging.mi-app.com"}
