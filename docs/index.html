¿Qué es Voxen?

Voxen es un orquestador de builds y capa de supply chain para Docker. Unifica la construcción, empaquetado, firma, generación de SBOM, dockerizado y publicación en una sola interfaz de línea de comandos.

🔧

Multi-Lenguaje

Funciona con Java, Kotlin, Node.js, Python, Go, Rust y .NET mediante un sistema de plugins. Detecta automáticamente el runtime de tu proyecto.

📦

Empaquetado Universal

Empaqueta como JAR, WAR, ZIP, wheel, DLL o binario nativo — todo con un solo comando: voxen package.

🐳

Dockeriza Automático

Genera imágenes Docker óptimas y mínimas con un solo comando. voxen dockerize produce imágenes multi-architecture con SBOM, checksums y firma embebidos.

🔒

Supply Chain Completa

Firma GPG, checksums SHA-256/512/MD5, SBOM CycloneDX, SLSA provenance, y etiquetas OCI — todo generado automáticamente en cada build.

🌐

Multi-Idioma

Soporte nativo para 6 runtimes con imágenes base óptimas para cada uno: distroless, alpine slim, y más. Plugins para Java, Node, Python, Go, Rust y .NET.

🚀

Pipeline CI/CD Listo

Workflows oficiales de GitHub Actions, un comando --json para integración con cualquier pipeline, y acción reutilizable voxen-build-push.

¿Cómo funciona?

Voxen sigue el ciclo de vida completo: desde el código fuente hasta la imagen Docker en el registry, pasando por calidad, seguridad y trazabilidad.

  1. Inicializa tu proyecto — voxen init --runtime java crea un voxen.json con validación automática contra JSON Schema.
  2. Construye — voxen build detecta el build tool (Maven, Gradle, npm, cargo, etc.) y lo ejecuta. Cada operación queda registrada en el audit log.
  3. Empaqueta — voxen package produce el artefacto distribuible. Se generan automáticamente checksums SHA-256/512/MD5, firma GPG, y SBOM CycloneDX.
  4. Dockeriza — voxen dockerize --tag ghcr.io/user/app:1.0 --push genera una imagen Docker mínima (distroless), multi-architecture, firmada, con labels OCI y SBOM embebido.
  5. Publica — voxen publish o voxen publish:docker sube tu artefacto o imagen a Maven Central, Artifactory, Nexus, S3, Docker Hub, GHCR, ECR o GCR.
  6. Despliega — voxen deploy --profile prod despliega con configuración específica del entorno. Perfiles integrados para dev, staging y producción.

Ejemplo Rápido

De una app Java a una imagen Docker firmada en 3 comandos.

# 1. Crear e inicializar un proyecto
mkdir mi-app && cd mi-app
voxen init -n mi-app -r java -t jar -o build

# 2. Construir y dockerizar
voxen build
voxen dockerize --tag ghcr.io/usuario/mi-app:1.0.0 --lint

# 3. Publicar imagen multi-architecture firmada
voxen dockerize --tag ghcr.io/usuario/mi-app:1.0.0 \
  --push --platform linux/amd64,linux/arm64 --sign --json
✅ La imagen resultante contiene: SBOM CycloneDX, checksums SHA-256/512/MD5, firma GPG, labels OCI estándar, y metadatos completos del build — todo sin escribir ni una línea de Dockerfile.

¿Por qué Voxen + Docker?

🎯

Menos conceptos

6 comandos vs. 9+ de Docker + Cosign + Trivy + Syft. Un solo tool reemplaza todo el stack de supply chain.

📏

Imágenes mínimas

Distroless por defecto. Sin shells, sin package managers, sin herramientas de build en la imagen final. Menor superficie de ataque.

📋

Trazabilidad nativa

Cada imagen incluye labels con build-id, perfil, runtime, checksums y SBOM. docker inspect te dice todo lo que necesitas saber.

🛡️

Seguridad por defecto

Firma GPG, Cosign keyless, escaneo de vulnerabilidades Trivy, y SLSA provenance — todo configurado en el pipeline de CI/CD.

🔌

Extensible por plugins

Añade un nuevo lenguaje implementando la interfaz RuntimePlugin o un nuevo publicador extendiendo Publisher.

🤝

Aliado de Docker

Voxen no compite con Docker. Lo potencia: build → dockerize → push. Docker se queda con lo que hace mejor (runtime aislado).
