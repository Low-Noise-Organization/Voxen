Referencia de Configuración

Voxen usa voxen.json para la configuración del proyecto y voxen-{perfil}.json para perfiles de entorno.

Esquema de voxen.json

El archivo de configuración usa JSON Schema (draft 2020-12) para validación. Cada proyecto debe tener name y runtime obligatoriamente.

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "name": {
      "type": "string",
      "description": "Nombre del proyecto",
      "minLength": 1,
      "maxLength": 128,
      "pattern": "^[a-zA-Z0-9_\\-.]+$"
    },
    "runtime": {
      "type": "string",
      "description": "Runtime del lenguaje",
      "minLength": 1,
      "maxLength": 32
    },
    "target": {
      "type": "string",
      "description": "Formato de salida (jar, war, zip, tar.gz, wheel)",
      "maxLength": 32
    },
    "output": {
      "type": "string",
      "description": "Directorio de salida relativo a la raíz del proyecto",
      "maxLength": 256,
      "pattern": "^[a-zA-Z0-9_\\-./]+$"
    }
  },
  "required": ["name", "runtime"],
  "additionalProperties": false
}

Campos

name (obligatorio)

Nombre del proyecto. Se usa en nombres de artefactos, coordenadas de publicación e identificadores de despliegue.

Tipo: string | Mín: 1 | Máx: 128 | Patrón: ^[a-zA-Z0-9_\-\.]+$

runtime (obligatorio)

Runtime del lenguaje. Determina qué plugin se usa para build, package, publish, deploy y dockerize.

Tipo: string | Máx: 32 | Ejemplos: java, kotlin, rust, node, python, go, dotnet

target (opcional)

Formato de empaquetado. El plugin lo usa para determinar el tipo de artefacto a generar.

Tipo: string | Máx: 32 | Ejemplos: jar, war, zip, tar.gz, wheel

output (opcional)

Directorio de salida para los artefactos, relativo a la raíz del proyecto. Por defecto: dist.

Tipo: string | Máx: 256 | Patrón: ^[a-zA-Z0-9_\-\.\/]+$ | Default: dist

Ejemplos por Lenguaje

Java / Kotlin

{
  "name": "mi-libreria",
  "runtime": "java",
  "target": "jar",
  "output": "target"
}

Node.js

{
  "name": "mi-app-web",
  "runtime": "node",
  "target": "zip",
  "output": "dist"
}

Python

{
  "name": "mi-servicio",
  "runtime": "python",
  "target": "wheel",
  "output": "dist"
}

Go / Rust / .NET

{
  "name": "mi-binario",
  "runtime": "go",
  "target": "bin",
  "output": "build"
}

Perfiles de Entorno

Los perfiles permiten definir variables y settings específicos por entorno. Perfiles integrados: dev, staging, prod.

Perfiles Integrados

PerfilVariablesSettings
devenv=development, optimization=none, debug=trueverbose=true, skipTests=false
stagingenv=staging, optimization=standard, debug=falseverbose=false, skipTests=false
prodenv=production, optimization=aggressive, debug=falseverbose=false, skipTests=true

Perfil Personalizado

Crea un archivo voxen-{nombre}.json en la raíz del proyecto:

// voxen-custom.json
{
  "name": "custom",
  "extends": "dev",
  "variables": {
    "env": "staging",
    "optimization": "standard",
    "db_url": "jdbc:postgresql://staging-db:5432/app"
  },
  "settings": {
    "skipTests": "false",
    "verbose": "false"
  }
}

Usa el perfil con el flag -p o --profile:

voxen build --profile custom
voxen deploy -p prod --json
voxen dockerize --tag ghcr.io/user/app:1.0 --profile staging

Credenciales

Las credenciales para registros y repositorios se guardan en ~/.voxen/credentials.json:

{
  "docker": {
    "username": "mi-usuario",
    "token": "ghp_xxxxxxxxxxxx"
  },
  "maven-central": {
    "token": "maven_xxxxxxxxxxxx"
  },
  "artifactory": {
    "username": "admin",
    "password": "pass123",
    "host": "https://artifactory.ejemplo.com"
  }
}

También se pueden configurar mediante variables de entorno:

export DOCKER_TOKEN="ghp_xxxxxxxxxxxx"
export MAVEN_CENTRAL_TOKEN="maven_xxxxxxxxxxxx"
