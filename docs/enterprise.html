Visión General

🐳 Dockerize — Orquestador de Build + Supply Chain NUEVO

Voxen no compite con Docker. Se sienta delante de Docker y lo potencia: genera imágenes Docker mínimas y óptimas con toda la metadata de supply chain embebida.

# Pipeline completo: build → dockerize → push
voxen build --profile prod
voxen dockerize --tag ghcr.io/user/app:1.0.0 --push \
  --platform linux/amd64,linux/arm64 --lint --sign

📋 Trazabilidad Completa

Cada operación queda registrada en un audit log estructurado en .voxen/audit.jsonl.

{"timestamp":"2026-07-10T12:00:00Z","operation":"build","project":"mi-app","status":"success","detail":"durationMs=3250"}
{"timestamp":"2026-07-10T12:01:00Z","operation":"dockerize","project":"mi-app","status":"success","detail":"tag=ghcr.io/user/app:1.0, push=true"}
{"timestamp":"2026-07-10T12:02:00Z","operation":"publish","project":"mi-app","status":"success","detail":"repository=maven-central"}

Uso programático

AuditLogger audit = new AuditLogger(projectDir);
audit.build("mi-app", true, 3250);
audit.publish("mi-app", true, "maven-central");
audit.deploy("mi-app", true, "staging");

📦 Supply Chain Seguridad

Firma GPG de artefactos

SigningService signer = new SigningService("key-id", "passphrase");
signer.sign(Path.of("build/mi-app.jar"));
// Genera: mi-app.jar.asc

Checksums automáticos

ChecksumGenerator checksums = new ChecksumGenerator();
checksums.generate(Path.of("build/mi-app.jar"));
// Genera: .sha256, .sha512, .md5

SBOM CycloneDX

SbomGenerator sbom = new SbomGenerator();
sbom.generate(projectConfig);
// Genera: build/bom.json

SLSA Provenance

SlsaProvenance slsa = new SlsaProvenance();
slsa.write(Path.of("build/provenance.json"), metadata);

🔒 Firma de Imágenes Docker

Cosign (keyless — recomendado)

ManifestSigner signer = new ManifestSigner();
signer.signWithCosignKeyless("ghcr.io/user/app:1.0.0");
# Usa Sigstore (sin necesidad de gestionar claves)

Cosign (key-based)

signer.signWithCosign("ghcr.io/user/app:1.0.0", "cosign.key");

Docker Content Trust (GPG)

signer.signWithGpg("ghcr.io/user/app:1.0.0", "KEY123", "passphrase");

Verificación

signer.verify("ghcr.io/user/app:1.0.0");

🔍 Escaneo de Vulnerabilidades

Integración con Trivy para escanear imágenes Docker en busca de CVEs:

VulnerabilityScanner scanner = new VulnerabilityScanner();
VulnerabilityScanner.ScanResult result = scanner.scan("ghcr.io/user/app:1.0.0");

if (!result.passed()) {
    System.err.println("¡Vulnerabilidades críticas encontradas!");
    result.vulnerabilities().forEach(v ->
        System.out.println(v.scanner() + ": " + v.severity()));
}

🔍 Linter de Imágenes

Analiza la imagen Docker generada y reporta problemas de calidad:

ImageLinter linter = new ImageLinter();
ImageLinter.LintResult result = linter.lint("ghcr.io/user/app:1.0.0");

// Errores: imagen no encontrada
// Advertencias:
//   - Tamaño excesivo (>500MB)
//   - Demasiadas capas (>10)
//   - Ejecución como root
//   - Sin puertos expuestos
//   - Sin etiquetas
//   - SBOM faltante

🏢 Multi-Entorno

Tres perfiles integrados (dev, staging, prod) más perfiles personalizados mediante archivos voxen-{nombre}.json.

voxen build --profile dev        # debug=true, verbose=true
voxen dockerize --profile staging  # sin debug, optimización estándar
voxen deploy -p prod --json       # salta tests, optimización agresiva

🛡️ Plugin Sandboxing

Aislamiento de plugins mediante Java SecurityManager. Restringe acceso a archivos, red y propiedades del sistema.

PluginSandbox sandbox = new PluginSandbox(true);
sandbox.applyIfEnabled("mi-plugin", Path.of("/ruta/al/plugin"));

📊 Telemetría (opt-in)

Métricas locales de operaciones, tiempos de ejecución y tasas de éxito/fallo. Almacenadas en ~/.voxen/telemetry.jsonl.

TelemetryCollector telemetry = new TelemetryCollector(true); // opt-in

String opId = "build-" + System.currentTimeMillis();
telemetry.startOperation(opId, "build", "mi-app");
// ... realizar build ...
telemetry.finishOperation(opId, true, "durationMs=3250");

System.out.println("Total: " + telemetry.totalOperations());
System.out.println("Fallos: " + telemetry.failedOperations());

🤖 CI/CD Integrado

Workflows oficiales de GitHub Actions y acción reutilizable:

# Usando la acción oficial
- uses: ./.github/actions/voxen-build-push
  with:
    profile: prod
    tag: ghcr.io/${{ github.repository }}:${{ github.ref_name }}
    push: true
    platforms: linux/amd64,linux/arm64
    sign: true
# Workflow completo con escaneo y firma
name: Build & Dockerize
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with: { java-version: '21', distribution: 'temurin' }
      - run: ./gradlew build
      - run: |
          voxen build --profile prod --json
          voxen dockerize \
            --tag ghcr.io/${{ github.repository }}:latest \
            --push --platform linux/amd64,linux/arm64 \
            --lint --sign --json
      - uses: aquasecurity/trivy-action@master
        with:
          image-ref: ghcr.io/${{ github.repository }}:latest
