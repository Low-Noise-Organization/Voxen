Guía de Plugins

Voxen usa dos sistemas de plugins: RuntimePlugin para detección de lenguajes y Plugin (SPI) para build/publicación. También puedes crear publicadores personalizados.

Plugins de Runtime (Multi-Lenguaje)

Los RuntimePlugin detectan automáticamente el lenguaje del proyecto y proporcionan el comando de build y la imagen Docker base.

Interfaz RuntimePlugin

public interface RuntimePlugin {
    String runtime();                           // nombre del lenguaje
    boolean detect(Path projectDir);            // detecta si aplica a este proyecto
    String buildCommand(Path projectDir);       // comando para construir
    String defaultArtifactPattern();            // patrón de artefactos generados
    String buildImage();                        // imagen base Docker
}

Añadir un RuntimePlugin nuevo

public class RubyRuntimePlugin implements RuntimePlugin {
    @Override
    public String runtime() { return "ruby"; }

    @Override
    public boolean detect(Path projectDir) {
        return projectDir.resolve("Gemfile").toFile().exists();
    }

    @Override
    public String buildCommand(Path projectDir) {
        return "bundle exec rake build";
    }

    @Override
    public String defaultArtifactPattern() {
        return "pkg/*.gem";
    }

    @Override
    public String buildImage() {
        return "ruby:3.3-slim";
    }
}

Plugins de Build (SPI)

Los plugins de build implementan la interfaz Plugin y se registran via ServiceLoader. Controlan el ciclo de vida completo: build, package, publish, deploy.

Interfaz Plugin

public interface Plugin {
    String name();
    String language();
    Set<PluginType> supportedTypes();
    default String apiVersion() { return "1.0"; }

    void init(PluginContext context);
    BuildResult build(PluginContext context);
    PackageResult packageArtifact(PluginContext context);
    PublishResult publish(PluginContext context);
    DeployResult deploy(PluginContext context);
}

Crear un Plugin de Build

  1. Crea un módulo Java que dependa de voxen-plugins-api
  2. Implementa la interfaz Plugin con los 5 métodos del ciclo de vida
  3. Regístrate via ServiceLoader: crea META-INF/services/io.lownoise.voxen.plugins.api.Plugin con el nombre completo de tu clase
  4. Compila y prueba: java -cp voxen.jar:tu-plugin.jar io.lownoise.voxen.cli.VoxenCli info

Resultados del Ciclo de Vida

Cada método del plugin devuelve un tipo de resultado específico:

MétodoResultadoCampos Clave
build()BuildResultsuccess, artifacts (List<Path>), durationMs, output
packageArtifact()PackageResultsuccess, packagePath, format, sizeBytes, output
publish()PublishResultsuccess, targetRepository, artifactUrl, output
deploy()DeployResultsuccess, targetEnvironment, deploymentUrl, output

Métodos factoría para resultados:

// Éxito
BuildResult.success(List.of(paths), durationMs, "mensaje")
PackageResult.success(path, "jar", fileSize, "mensaje")
PublishResult.success("maven-central", "https://...", "mensaje")
DeployResult.success("staging", "https://...", "mensaje")

// Fallo
BuildResult.failure("Mensaje de error")
PackageResult.failure("Mensaje de error")
PublishResult.failure("Mensaje de error")
DeployResult.failure("Mensaje de error")

Publicadores Personalizados

Además de los 6 publicadores incluidos (local, maven-central, artifactory, nexus, s3, docker), puedes crear los tuyos:

public class GitHubPackagesPublisher implements Publisher {
    @Override
    public String name() { return "github-packages"; }

    @Override
    public PublishResult publish(Path artifact, String repository,
                                  String groupId, String artifactId, String version) {
        // Implementar subida a GitHub Packages
        return PublishResult.success(repository, url, "Published to GitHub Packages");
    }
}

Los publicadores se auto-registran vía PublisherFactory.register() al estar en el classpath.

PluginContext API

Los plugins reciben un objeto PluginContext con información del entorno:

public interface PluginContext {
    Path workingDirectory();     // Raíz del proyecto
    Path outputDirectory();      // Directorio de salida (default: dist)
    boolean verbose();           // Flag --verbose
    boolean debug();             // Flag --debug
    void log(String message);    // Salida normal
    void verbose(String msg);    // Salida verbose
    void debug(String msg);      // Salida debug
    void error(String message);  // Salida de error
}
